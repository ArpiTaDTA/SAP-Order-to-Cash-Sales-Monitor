Problem Statement

Business users often need multiple SAP transactions and reports to track sales-order progress across the Order-to-Cash cycle. This makes it hard to identify pending delivery, billing delays, and overdue payments in a single place.

Solution Overview

This project provides a SAP Fiori style dashboard built with SAPUI5/OpenUI5. It uses a local dataset inspired by the ZSODATA ABAP report and presents sales order data through KPI cards, filters, search, and a detailed drill-down page.

Features

  • KPI dashboard for total orders, order value, customers, and pending actions
  • Search by sales order, customer, or material
  • Filter by process stage, sales organization, and payment status
  • Detailed sales order page with business and technical mapping
  • Responsive UI for desktop and mobile-friendly usage
  • OData-ready metadata reference for future SAP integration

Technology Stack

  • SAPUI5 / OpenUI5
  • JavaScript
  • XML Views
  • JSON Model
  • OData metadata reference
